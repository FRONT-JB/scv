# 저장소 작업 지침

## 커밋 메시지

이 저장소의 새 커밋과 커밋 히스토리 재작성은 Conventional Commits 형식을 따른다.

```text
<type>(<scope>): <한국어 요약>

<선택적인 한국어 본문>
```

- `type`은 `feat`, `fix`, `docs`, `style`, `refactor`, `perf`, `test`, `build`, `ci`, `chore`, `revert` 중에서 고른다.
- `scope`는 선택 사항이며 `repo`, `plugin`, `readme`, `e2e`처럼 짧은 영문 소문자로 작성한다.
- 제목의 설명은 한국어로 작성하고 72자 이내로 유지하며 마침표를 붙이지 않는다.
- 고유명사, 파일 경로, 명령어, 코드 식별자는 필요한 경우 원문 표기를 유지한다.
- 본문이 필요하면 제목 다음에 빈 줄을 두고 한국어 `-한다`체로 변경 내용과 이유를 설명한다.
- 하나의 커밋에는 하나의 논리적 변경만 담고, 관련 없는 변경은 별도 커밋으로 나눈다.
- 실제 공동 작성자가 있고 사용자가 명시적으로 요청한 경우가 아니면 `Co-Authored-By`를 추가하지 않는다. 도구나 에이전트 계정을 공동 작성자로 자동 기록하지 않는다.
- 커밋 작성자 이름과 이메일은 실제 작업자의 GitHub 계정에 연결된 값을 사용하며 자동화가 임의로 바꾸지 않는다.
- 히스토리 재작성과 강제 푸시는 사용자의 명시적 승인을 받은 뒤 `--force-with-lease`로 실행한다.

예시:

```text
feat(plugin): 실시간 실행 진행 조회 추가
fix(state): 재개 시 상태 인덱스 검증 오류 수정
docs(readme): 설치 절차 보강
test(e2e): 실패 후 재시도 흐름 검증
chore(plugin): Codex 캐시 무효화 버전 갱신
```
