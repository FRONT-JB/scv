# 알려진 품질 공백

테스트, 보안, 성능, 접근성, 운영과 문서에서 확인된 품질 공백을 관리한다. 공백을
숨기거나 성공으로 간주하지 않고 영향, 임시 완화와 해소 조건을 명시한다.

## 현재 공백

현재 등록된 공백이 없다. 프로젝트 조사 또는 검증에서 공백이 확인되면 아래
형식으로 추가한다.

| ID | 영역 | 영향 | 근거 | 임시 완화 | 담당 | 해소 조건 | 상태 |
|---|---|---|---|---|---|---|---|

## 관리 규칙

- ID는 프로젝트 안에서 안정적으로 참조할 수 있게 유지한다.
- 영향과 근거를 확인하지 않은 추측은 조사 문서에 먼저 기록한다.
- 보안 경계 위반, 데이터 손실 가능성, 승인 범위 이탈과 결정적 검증 실패는 단순
  문서화만으로 수용하지 않는다.
- 공백을 수용하려면 관련 finding, 영향, 보완 조치, 담당, 만료 조건과 사용자 승인
  revision을 연결한다.
- 공백이 해소되어도 항목을 삭제하지 않고 해소 근거와 상태를 갱신한다.
