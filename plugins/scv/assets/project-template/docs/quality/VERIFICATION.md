# 검증 기준

이 문서는 프로젝트 변경을 검증하는 계층과 표준 실행 방법의 권위 원본이다.
실제 명령은 저장소에서 재현 가능하다고 확인한 뒤 아래 표에 등록한다.

## 검증 계층

1. 형식, schema, lint와 같은 결정적 정적 검사
2. 변경 영역에 직접 관련된 테스트
3. 프로젝트 전체 회귀 테스트
4. 안전 경계와 실패 경로 검사
5. Codex의 읽기 전용 최종 코드 리뷰

상위 계층이 하위 계층을 대신하지 않는다. 적용할 수 없는 계층은 생략 이유와
남는 위험을 핸드오프에 기록한다.

## 표준 명령

| 목적 | 명령 | 필요한 환경 | 성공 조건 |
|---|---|---|---|
| 결정적 정적 검사 | 프로젝트 확인 후 등록 | 최소·비밀정보 없음 | 종료 코드와 출력 기준을 명시 |
| 관련 테스트 | 프로젝트 확인 후 등록 | 네트워크 차단 | 변경 영역의 테스트 통과 |
| 전체 회귀 | 프로젝트 확인 후 등록 | 네트워크 차단 | 전체 suite 통과 |

명령에는 네트워크 접근, 자격증명 또는 장비별 경로를 전제로 하지 않는다. 필요한
입력과 생성 파일은 승인된 경로와 격리된 scratch 안으로 제한한다.

## 증거 기록

검증마다 다음 정보를 남긴다.

- 검증 목적과 적용 범위
- 실행한 명령의 정규화된 형태
- 종료 코드, 실행 시간과 결과 요약
- 대상 workspace checkpoint와 산출물 digest
- 실패 또는 생략 사유와 후속 조치

비밀값, 환경 전체, 인증 파일과 제한을 넘는 원시 출력은 증거에 저장하지 않는다.

## 갱신 조건

테스트 runner, 검증 명령, 필수 도구, 성공 조건 또는 지원 환경이 바뀌면 같은
작업에서 이 문서를 갱신하고 실제 명령으로 재검증한다.
