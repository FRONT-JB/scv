# 변경 이력

사용자에게 의미 있는 동작, 호환성, 설정과 운영 방식의 변경을 기록한다. 내부
정리만으로 외부 동작이 달라지지 않는 경우에는 기록하지 않는다.

## 작성 규칙

- 아직 배포되지 않은 변경은 `미출시` 아래에 누적한다.
- 항목은 `추가`, `변경`, `수정`, `보안`, `제거` 중 알맞은 범주에 둔다.
- 관련 작업 snapshot이나 ADR이 있으면 상대 경로로 연결한다.
- 사용자에게 필요한 전환 또는 복구 절차를 변경 항목과 함께 설명한다.
