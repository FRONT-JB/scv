# 시스템 개요

이 문서는 프로젝트의 시스템 경계, 구성요소와 주요 데이터 흐름을 설명하는
권위 있는 개요다. SCV가 최초 구성을 만들 때 기본 골격으로 제공하며, 실제
프로젝트 구조를 조사한 뒤 확인된 내용으로 갱신한다.

## 시스템 목적

- 프로젝트가 해결하는 사용자 문제와 핵심 책임을 기록한다.
- 프로젝트가 직접 소유하는 범위와 외부 시스템의 책임을 구분한다.
- 구현 변경을 검토할 때 유지해야 할 불변식과 품질 속성을 명시한다.

## 시스템 경계

다음 항목을 프로젝트 사실에 맞게 기록한다.

- 진입점과 사용자 또는 상위 시스템
- 프로젝트가 소유하는 데이터와 상태
- 외부 서비스, 저장소와 실행 환경
- 인증, 권한, 개인정보와 보안 경계
- 명시적으로 지원하지 않는 사용 방식

추측을 사실처럼 기록하지 않는다. 확인되지 않은 내용은 근거와 함께
`KNOWN-GAPS.md` 또는 조사 문서에 남긴다.

## 구성요소

구성요소마다 책임, 입력, 출력, 의존성, 실패 방식을 짧게 정리한다. 소스 경로와
설정 이름은 실제 저장소에서 확인한 값만 사용한다.

## 주요 데이터 흐름

사용자 요청이나 외부 입력이 시스템에 들어온 뒤 검증, 처리, 저장되고 결과로
나가는 흐름을 설명한다. 동기·비동기 경계, 재시도와 오류 복구가 있다면 함께
기록한다.

## 운영과 품질 경계

배포, 관측, 장애 대응, 성능과 보안에 영향을 주는 프로젝트 고유 규칙을 기록한다.
검증 명령과 알려진 공백은 각각 `quality/VERIFICATION.md`와
`quality/KNOWN-GAPS.md`에서 관리한다.

## 갱신 조건

다음 변경이 승인되면 같은 작업에서 이 문서를 검토한다.

- 시스템 경계나 주요 책임의 변경
- 새 외부 의존성 또는 데이터 흐름의 도입
- 인증, 권한, 저장 또는 실패 복구 방식의 변경
- 기존 설명을 무효화하는 구조 변경
